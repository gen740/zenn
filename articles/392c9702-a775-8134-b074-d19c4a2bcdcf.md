Neovim 0.13 の `vim.ui.img` を使って、TeX の Live Preview を作りました。左側で TeX ソースを書き、右側に PDF のプレビューを表示します。保存すると `latexmk` で再コンパイルし、`<CR><CR>` で SyncTeX forward search を実行します。
この記事は、`vim.ui.img` を Neovim の UI 部品として使う実装例です。

# 作ったもの

作ったものは、Neovim の中で完結する TeX Live Preview です。

* 右側の split に PDF ページを表示する
* `*.tex` の保存時に `latexmk` で再コンパイルする
* コンパイル後にプレビューを更新する
* `<CR><CR>` でカーソル位置に対応する PDF ページへ移動する
* 対応箇所を半透明の矩形でハイライトする
* ウィンドウのリサイズに追従する

PDF は直接表示しません。必要なページだけを `pdftoppm` で PNG に変換し、その画像を `vim.ui.img` で表示します。

# なぜ作ったか

これまでは Vim と Skim を組み合わせて TeX を書いていました。Skim に大きな不満があったわけではありません。ただ、エディタと PDF ビューアが別アプリだと、アプリケーションの切り替えとウィンドウ配置が発生します。
Neovim の split として PDF を扱えれば、編集、保存、コンパイル、プレビュー、対応箇所へのジャンプを同じ環境で完結できます。

# 全体の流れ

処理の流れはこうです。

```plain text
TeX source
  ↓ save
latexmk
  ↓
main.pdf
  ↓
pdftoppm
  ↓
PNG
  ↓
vim.ui.img
  ↓
right split
```

SyncTeX では、カーソル位置から PDF 上の対応箇所を問い合わせます。

```plain text
cursor position
  ↓
synctex view
  ↓
page and box
  ↓
convert coordinates
  ↓
vim.ui.img overlay
```

# PDF ページを右ペインに表示する

起動時に、現在の TeX ソースを開いているウィンドウを覚え、右側に preview 用の split を作ります。

```lua
S.source_win = vim.api.nvim_get_current_win()

vim.cmd('rightbelow vsplit')
S.preview_win = vim.api.nvim_get_current_win()
S.preview_buf = vim.api.nvim_create_buf(false, true)
vim.api.nvim_win_set_buf(S.preview_win, S.preview_buf)

vim.bo[S.preview_buf].bufhidden = 'wipe'
vim.bo[S.preview_buf].buftype = 'nofile'

vim.api.nvim_set_current_win(S.source_win)
```

表示するページは `pdftoppm` で PNG に変換します。

```lua
vim.system({
  'pdftoppm', '-png',
  '-f', tostring(page), '-l', tostring(page),
  '-singlefile',
  '-scale-to-x', tostring(target_px),
  '-scale-to-y', '-1',
  S.pdf, S.prefix,
}, { text = false }, function(res)
  -- PNG を読み込んで vim.ui.img.set する
end)
```

変換後の PNG を `vim.ui.img.set` に渡します。

```lua
local ok, id = pcall(vim.ui.img.set, blob, {
  row = pr,
  col = pc,
  width = w,
  height = h,
  zindex = cfg.zindex,
})
```

# セルの縦横比を補正する

ターミナルの 1 セルは正方形ではありません。そのため、画像のピクセル比率をそのままセル数へ変換すると、PDF が歪みます。
実装ではセルの縦横比を取得し、表示サイズを補正します。この補正を入れないと、PDF の表示だけでなく SyncTeX のハイライト位置もずれます。

```lua
local function fit_cells(img_w, img_h, cols, rows)
  local unit_w = img_w
  local unit_h = img_h / cfg.cell_aspect
  local scale = math.min(cols / unit_w, rows / unit_h)
  local w = math.max(1, math.floor(unit_w * scale))
  local h = math.max(1, math.floor(unit_h * scale))
  return w, h
end
```

# 保存時にコンパイルする

TeX ファイルを保存したら `latexmk` を呼びます。

```lua
vim.api.nvim_create_autocmd('BufWritePost', {
  group = S.augroup,
  pattern = '*.tex',
  callback = function(args)
    local f = vim.api.nvim_buf_get_name(args.buf)
    if f:sub(1, #S.root) == S.root then
      compile()
    end
  end,
})
```

コンパイルに成功したら、現在のページを再描画します。保存を連続で行ったときにコンパイルが重ならないように、`compiling` と `pending` も使っています。

```lua
if S.compiling then
  S.pending = true
  return
end
```

# SyncTeX で対応箇所へ移動する

`<CR><CR>` に SyncTeX forward search を割り当てます。

```lua
vim.keymap.set('n', '<CR><CR>', sync_from_cursor, {
  buffer = buf,
  silent = true,
  desc = 'auto_preview: SyncTeX forward search',
})
```

カーソル位置から行番号、列番号、ファイル名を取り出し、`synctex view` に渡します。

```lua
vim.system({
  'synctex', 'view',
  '-i', string.format('%d:%d:%s', line, colnr, file),
  '-o', S.pdf,
}, { text = true }, function(res)
  -- 出力から page と box を読む
end)
```

ページを表示したあと、box があればその位置をハイライトします。

```lua
show(tonumber(page), box and function()
  place_highlight(box)
end or nil)
```

# ハイライトは画像を重ねる

ハイライトも `vim.ui.img` で表示します。半透明の 1x1 PNG を作り、SyncTeX が返した矩形に合わせて拡大します。
つまり、PDF ページ画像の上に、別の画像を overlay として重ねています。必要なのは、PDF 座標からターミナルのセル座標への変換です。

# まとめ

Neovim 0.13 の `vim.ui.img` を使って、TeX Live Preview を Neovim 内に実装しました。
PDF ページを PNG に変換して右ペインに表示し、SyncTeX と組み合わせて対応箇所へ移動します。ハイライトも画像 overlay として重ねています。
外部 PDF ビューアを置き換えるものではありません。それでも、執筆中に「今のソースが PDF 上でどう見えるか」を確認する用途なら十分使えます。
