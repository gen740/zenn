# HEASoft 環境を Nix flake にした

HEASoft の環境構築には時間がかかります。ビルドに数時間かかることがあり、途中で止まると復旧も面倒です。
使い始めるにも手順があります。`HEADAS` を設定し、`headas-init.sh` を読み込み、CALDB も設定します。XMM-SAS を使う場合は `SAS_DIR` と `sas-setup.sh` も必要です。
この手順を解析ごとに手で管理したくなかったので、HEASoft、XMM-SAS、CALDB を Nix flake にまとめました。
目的は 2 つです。
1 つ目は、環境構築を簡単にすることです。
2 つ目は、解析環境を再現しやすくすることです。
解析では、software version と CALDB が結果に関わります。査読対応で解析をやり直すときも、当時の環境を再現できる必要があります。

# flake module として分けた

`heasoft`、`xmmsas`、`caldb` をそれぞれ flake-parts module として分けました。
repository をそのまま build できます。自分の解析用 flake に module として取り込むこともできます。
デフォルトの source で使う場合は `flakeModules` を使います。上流の URL や hash を差し替える場合は `mkFlakeModules` を使います。

# devShell で初期化まで済ませる

この flake では、devShell に入った時点で解析環境を初期化できるようにしました。
利用側では、`devShells.default` の `packages` に次の package を並べます。

* `config.packages.heasoft`
* `config.packages.xmmsas`
* `config.packages.caldb`

そのうえで `nix develop` に入ります。
HEASoft の version は `fversion` で確認できます。
CALDB は `echo $CALDB` で確認できます。
利用側で `shellHook` を書く必要はありません。各 package に `nix-support/setup-hook` を持たせています。
HEASoft の hook は `HEADAS` を設定し、`headas-init.sh` を読み込みます。CALDB の hook は `CALDB` を設定し、`caldbinit.sh` を読み込みます。XMM-SAS の hook は `SAS_DIR` を設定し、`sas-setup.sh` を読み込みます。
HEASoft を解析に使うには、binary だけでなく環境変数と初期化スクリプトが必要です。その初期化を Nix 側に寄せました。
macOS で XSPEC を動かすには XQuartz も必要です。X11 を使う部分があるため、Nix 側の package だけで完結しない点に注意してください。

# CALDB を解析ごとに固定できるようにした

CALDB は setup files に加えて、任意の calibration archive を追加できるようにしました。
`caldb.nix` は `caldbSrc` と `caldbData` を受け取ります。
`caldbSrc` は CALDB setup files です。`caldbData` は追加で展開する calibration archive のリストです。
これで、解析で使う CALDB を flake の中に明示できます。
Nix では project ごとに異なる CALDB を指定できます。古い解析には古い CALDB を使い、新しい解析には新しい CALDB を使う構成を取りやすくなります。

# patch も必要だった

Nix 化では、依存関係を書くだけで済みませんでした。
HEASoft には古い C / C++ / Fortran / shell script が含まれます。Nix の build 環境で通らない箇所や、実行時に問題が出る箇所がありました。
XSPEC では、build 後の実行時に問題が出ました。そのため、一部の C++ code に patch を当てています。
Nix 化の過程で、どこに手当てが必要だったかを patch として残せました。

# binary cache は提供しない

この repository は、HEASoft、XMM-SAS、CALDB 自体を再配布しません。
それぞれを upstream から取得して build / install するための Nix packaging code です。
binary cache は提供しません。binary cache を提供すると、HEASoft、XMM-SAS、CALDB のビルド済み成果物を配布することになり、確実に再配布に当たるためです。
同じ理由で、現時点では nixpkgs に入れるつもりもありません。
ただし、再配布に関する制約を適切に回避でき、nixpkgs 側で受け入れ可能で、実際に利用者がいるなら、maintainer になる意思はあります。

# まとめ

HEASoft / XMM-SAS / CALDB を Nix flake としてまとめました。
環境構築を簡単にするためです。解析環境を再現しやすくするためです。
解析パイプラインを固定する前に、解析環境を固定する必要がありました。この flake は、そのための土台です。
